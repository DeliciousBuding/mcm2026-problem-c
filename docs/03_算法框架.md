## A1. 统一符号与“几何正确表述”（修补你最容易被挑刺的点）

**必须给的公式：**

**(A1) Fan share simplex：**
[
\mathcal{S}*{n}=\left{\mathbf v\in\mathbb R^n:\sum*{i=1}^n v_i=1,\ v_i\ge \epsilon\right}.

**(A2) 可行集不是 hyperrectangle：**

这句话能避免评委抓你“hyperrectangle 乘积独立”的小辫子。

## A2. Percent‑rule（LP 核心）怎么写（要短、硬、清楚）

你现在是：

* (C_{i,t}=\alpha Jpct_{i,t}+(1-\alpha)v_{i,t})
* eliminated has lowest combined score
* slack S* 诊断
  方向完全对。

**正文只保留 3 个关键公式：**

**(A3) Normalized judge share：**
[
j_{i,t}=\frac{J_{i,t}}{\sum_{k\in C_t}J_{k,t}}.
]

**(A4) Combined score：**
[
c_{i,t}(\alpha)=\alpha j_{i,t}+(1-\alpha)v_{i,t}.
]

**(A5) Elimination constraints（无 slack 版）：**
[
c_{E,t}(\alpha)\le c_{i,t}(\alpha),\quad \forall i\neq E.
]

**Slack 版（只写 1 行，不要展开太多）：**
[
\min_{\mathbf v,\mathbf s}\sum_i s_i\ \text{s.t.}\ (A1),(A5)\ \text{relaxed by }s_i\ge0.
]
并定义 (S^*=\min\sum s_i) 作为 **Assumption–Data Tension**。

### 这节要放的算法框（Algorithm Box 1）

**Algorithm 1（Percent Week Polytope Audit）**

1. 输入：当周选手集合 (C_t)，judge scores (J_{i,t})，淘汰者 (E)，参数 (\alpha,\epsilon)。
2. 构造约束 (A1)(A5) 得到 polytope (\mathcal P_t)。
3. 用 LP 求每个 (v_i) 的边界：
   [
   L_i=\min_{\mathbf v\in\mathcal P_t} v_i,\quad U_i=\max_{\mathbf v\in\mathcal P_t} v_i.
   ]
4. 计算 slack (S^*_t) 与可行性指标（见 A4）。
5. 输出：(\mathcal P_t) 的边界、可行性指标、样本接口。

> 注意：这里你“LP 求区间”是评委很喜欢的可解释内容（比黑箱回归强）。

## A3. Rank‑rule（MILP + 连续 share）写法：冲奖版更统一

题面 rank 规则例子很清楚：judge rank + fan rank 相加，最差者淘汰。

**冲奖版写法要做到两点：**

* 用 MILP 保证 rank 逻辑可行；
* 同时仍然输出 fan share（连续），满足题目“estimated fan votes”。

### (A6) Fan rank assignment MILP（你已有，保留但更简洁）

[
x_{ik}\in{0,1},\quad \sum_{k=1}^n x_{ik}=1,\quad \sum_{i=1}^n x_{ik}=1,

[
r^F_i=\sum_{k=1}^n kx_{ik}.
]

### (A7) 把 rank 与 share 连接（这是冲奖版关键）

用“大 M + 小间隔”把 fan rank 转成 share 序关系（只要一行就够）：
[
r^F_i<r^F_j \Rightarrow v_i\ge v_j+\Delta,

MILP 里用 big‑M 线性化（放 appendix 细节，正文只说“big‑M linearization”）。

### (A8) Combined rank & elimination（统一成“最差淘汰”）

设 judge rank (r^J_i)（由 (J_{i,t}) 排序得到）
combined rank：
[
R_i = r^J_i + r^F_i.
]
淘汰者满足：
[
R_E \ge R_i,\ \forall i\neq E.
]

### Judge‑Save 版（S28+）

题面说：先找 bottom two，再由 judges 选淘汰。
冲奖版写成两层：

1. bottom‑two 集合 (B_t) 来自 (R_i) 最大的两人
2. judges 在 (B_t) 里做选择：这一步用一个可学习参数 (\beta)（见 A6/Model B）。

### Algorithm Box 2

**Algorithm 2（Rank Season Feasible Orders → Feasible Shares）**

1. MILP 找到所有（或采样若干）可行 fan rank 排列 (\pi)。
2. 对每个 (\pi)，构造带序约束的 share polytope：(\mathcal P_t(\pi)=\mathcal S_n\cap {\text{order constraints}})。
3. Hit‑and‑Run 在 (\mathcal P_t(\pi)) 上采样得到 (v) 样本。
4. 混合所有 (\pi) 的样本得到该周 fan share 后验样本。

> 这比“只推 rank 不推 share”更符合题目，也比“硬把 rank 当 vote”更严谨。

你现在 XGBoost+SHAP 主要在预测生存/淘汰风险，虽然有 insight，但“对题”差一点。
冲奖版做法：**并列建两个可解释模型**（再用 XGBoost 当辅助）。

## C1. 建模对象：用 share 而不是 raw（避免跨周不可比）

* judges：(j_{i,t})
* fans：(\hat v_{i,t})（posterior mean 或 median；也可用样本做 Bayesian regression）

## C2. 模型形式（强烈推荐混合效应 / 层级模型）

### Judges 模型（J‑model）

[
\text{logit}(j_{i,t}) =
\mathbf x_i^\top\beta^{(J)} + u_{\text{pro}(i)}^{(J)} + u_{\text{season}(s)}^{(J)} + \epsilon_{i,t}.
]

### Fans 模型（F‑model）

[
\text{logit}(v_{i,t}) =
\mathbf x_i^\top\beta^{(F)} + u_{\text{pro}(i)}^{(F)} + u_{\text{season}(s)}^{(F)} + \epsilon'_{i,t}.
]

其中：

* (\mathbf x_i)：年龄、行业、（可选）地域等特征
* (u_{\text{pro}})：职业舞伴随机效应（你要回答“pro dancer 影响多大”）
* season 随机效应吸收不同季热度/规则差异

> 这套写法非常“实证论文范”，评委比看到 XGBoost 更信服因果/解释。

## C3. 你还可以保留 XGBoost（但放成“预测辅助”）

* 正文只放 forward‑chaining AUC 图（你现在 Fig.9 很好）
* SHAP summary 放 appendix（或正文只放 Top5）

## C4. Task3 的两张“拿奖图”

### 图王 C1：Pro dancer effect forest plot（最像顶刊）

* y：pro dancer 名单
* x：随机效应 (u_{\text{pro}}^{(J)})（上图）与 (u_{\text{pro}}^{(F)})（下图）
* 误差棒：95% CI

### 图王 C2：特征影响差异散点（Judge vs Fan）

# 十、Model D：DAWS（Task5）——让它不只是“方案”，而是“有性质的机制”

你现在 DAWS 已经有：动态 αt + 平滑约束 + Pareto + noise robustness。
冲奖版再加两块：**参数学习（judge‑save β）+ 性质（propositions）**。

## D1. DAWS 定义（你现有公式可保留，但解释更“机制设计”）

你当前：
[
\alpha_t = \text{clip}\Big(\alpha_0 + \gamma \frac{t}{T} - \eta U_t,\ \alpha_{\min},\alpha_{\max}\Big),\quad
|\alpha_t-\alpha_{t-1}|\le\delta.
]

冲奖版建议加一句：

* (U_t) = uncertainty index（例如平均 HDI 宽度或 entropy）
* 解释：不确定性高 → 增加 judge 权重以降低“随机淘汰”，不确定性低 → 给 fans 更多主权

## D2. 给 DAWS 两条“可证明/可解释性质”（很加分）

写成 **Proposition 1/2**（不需要很长证明，半页足够）：

### Proposition 1（Monotonicity）

若某选手的 judge share 和 fan share 同时上升，则其 DAWS score 不下降，淘汰风险不应变高。

### Proposition 2（Stability bound）

由于 (|\alpha_t-\alpha_{t-1}|\le\delta)，当周 score 的变化有上界：
[
|c_{i,t}-c_{i,t-1}| \le \delta|j_{i,t}-v_{i,t}| + (1-\alpha_t)|v_t-v_{t-1}| + \alpha_t|j_t-j_{t-1}|.
]
（写个“因此 DAWS 抑制过度摇摆”的结论即可。）

## D3. Judge‑save “softness β”不要只做敏感性，要做“学习”

题面说 judge save 有主观性；你现在用 β 做敏感性很好。
冲奖版升级为“估计 β”：

设 bottom‑two 为 ({a,b})，定义 judges 选择淘汰 a 的概率：
[
\Pr(E=a|{a,b})=\sigma\Big(\beta,(J_b-J_a)\Big),
]
其中 (\sigma(\cdot)) 为 logistic。
用 S28+ 数据（或你推断的 bottom‑two）做 MLE/贝叶斯估计，给出 (\hat\beta) 和 CI。

**图：Judge‑save decision curve**

* x：bottom‑two 的 judge score 差
* y：淘汰概率
* 画 logistic 曲线 + 置信带
  这张图“专业度爆表”。
