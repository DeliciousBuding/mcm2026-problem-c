## 开发流程（必须遵守）
- **主分支只读**：禁止在 `main` 上直接提交，所有改动在任务分支进行。
- **分支纯粹**：一个分支只做一个 Block，不跨 Block 修改；命名如 `block1-core-evidence`、`docs-cleanup`。
- **中文提交**：commit message 必须中文，描述“做了什么”；每次 commit 后立即 `git push`。
- **中断切换**：未完成需切分支时，用 `git stash push -u -m "中文说明"` 暂存，再切换处理。
- **验收后合并**：自检仅影响本 Block + 测试通过，再请求合并；合并后提示打 tag（手动执行）。

---

## 运行与编译（常用命令）
- **运行流水线**（根目录）：
  - `conda activate mcm2026`
  - `python -u src\pipeline.py`
- **仅运行 Synthetic Validation**（根目录）：
  - PowerShell：`$env:MCM_SYNTHETIC_ONLY="1"`
  - CMD：`set MCM_SYNTHETIC_ONLY=1`
  - `python -u src\pipeline.py`
- **编译英文/中文 PDF**（`paper/` 目录）：
  - `pdflatex -interaction=nonstopmode -halt-on-error main.tex`
  - `xelatex -interaction=nonstopmode -halt-on-error main_zh.tex`
